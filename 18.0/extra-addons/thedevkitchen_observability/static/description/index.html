🔍 TheDevKitchen Observability

OpenTelemetry instrumentation for distributed tracing and APM (Application Performance Monitoring)

OpenTelemetry 1.22+ Grafana Tempo OTLP gRPC W3C Trace Context

🎯 Features

🔗
Automatic HTTP Tracing
Controllers decorated with @trace_http_request automatically capture requests with full HTTP semantics (method, URL, status code, user info).
🌐
Distributed Context Propagation
W3C Trace Context support enables tracing across microservices and external systems.
📊
Grafana Integration
Seamless integration with Grafana Tempo for trace visualization, and Loki for log correlation.
🔗
Log Correlation
Automatic trace_id and span_id injection in logs enables one-click navigation from logs to traces in Grafana.
⚡
Performance Optimized
Batch span processor with configurable sampling (always_on, always_off, or ratio-based) for production use.

🚀 Quick Start

Step 1: Install Dependencies
pip install -r extra-addons/thedevkitchen_observability/requirements.txt
Step 2: Configure Environment
# Add to .env OTEL_ENABLED=true OTEL_SERVICE_NAME=odoo-production OTEL_EXPORTER_OTLP_ENDPOINT=tempo:4317 OTEL_TRACES_SAMPLER=always_on
Step 3: Install Module
Install this module via Odoo Apps interface or CLI:
docker compose exec odoo odoo -u thedevkitchen_observability -d realestate --stop-after-init
Step 4: Add Decorator
from odoo.addons.thedevkitchen_observability.services.tracer import trace_http_request @http.route('/api/v1/users', type='http', auth='none', methods=['GET']) @require_jwt @require_session @trace_http_request # 👈 Add this def list_users(self, **kwargs): return request.make_json_response({'users': [...]})

📖 Usage Examples

Controller Tracing

from odoo.addons.thedevkitchen_observability.services.tracer import trace_http_request @http.route('/api/v1/properties', type='http', auth='none', methods=['POST']) @require_jwt @require_session @trace_http_request def create_property(self, **kwargs): # Automatically traced with HTTP semantics # trace_id is in logs, linked to Grafana Tempo property_data = json.loads(request.httprequest.data) property = request.env['real.estate.property'].create(property_data) return request.make_json_response({'id': property.id})

Manual Instrumentation

from odoo.addons.thedevkitchen_observability.services.tracer import ( get_tracer, add_span_attribute, add_span_event, ) def complex_calculation(self, property_id): tracer = get_tracer() with tracer.start_as_current_span("calculate_property_value") as span: span.set_attribute("property.id", property_id) # Add events for milestones add_span_event("fetch.market.data.start") market_data = self._fetch_market_data() add_span_event("fetch.market.data.complete") # Calculate value = self._calculate_value(market_data) span.set_attribute("property.value", value) return value

🔗 Grafana Integration

Once configured, traces appear in Grafana Tempo with full correlation:

  • Metrics → Traces: Click exemplars in Prometheus graphs to see related traces
  • Logs → Traces: Click "View Trace" button in Loki to jump to Tempo
  • Traces → Logs: Click span in Tempo to see all related logs in Loki
# Example TraceQL queries in Tempo: # Find slow requests {duration>1s} # Find errors {service.name="odoo-production" && status=error} # Search by endpoint {http.route="/api/v1/users"}

⚙️ Configuration

Variable Default Description
OTEL_ENABLED true Enable/disable tracing
OTEL_SERVICE_NAME odoo Service identifier
OTEL_EXPORTER_OTLP_ENDPOINT tempo:4317 Tempo gRPC endpoint
OTEL_TRACES_SAMPLER always_on Sampling: always_on, always_off, traceidratio:0.1

🐛 Troubleshooting

⚠️ Traces not appearing?

1. Check logs: docker compose logs odoo | grep OpenTelemetry
2. Verify Tempo is running: docker compose -f docker-compose.observability.yml ps
3. Test connectivity: docker compose exec odoo nc -zv tempo 4317
4. Check environment variables are set correctly
